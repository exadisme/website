Today, I'd like to talk about my experience switching from Arch Linux to Gentoo.

First, let me mention that before installation, I set up an encrypted btrfs filesystem with the following subvolumes: "@", "@log", "@swap", "@snapshots", and "@home". Mounted as such:

    └─nvme0n1p2 259:2 0 465.4G 0 part
    └─root 254:0 0 465.4G 0 crypt
    @log        /var/log
    @swap       /swap
    @snapshots  /snapshots
    @home       /home
    @           /

One of the reasons I decided to switch to Gentoo was because of its flexibility and ability to be customized. This was particularly appealing to me because I wanted to try out different init systems. Initially, I experimented with open rc, but ultimately decided to revert back to systemd as I found it to be faster overall.

In Gentoo Linux, useflags are a way to enable or disable specific features in packages when they are compiled. There are two types of useflags: global and per-application.

Global useflags are defined in the /etc/portage/make.conf file and apply to all packages on the system. For example, if you wanted to enable support for XMPP in all packages that have it as an option, you could add "XMPP" to the USE variable in make.conf like this:

    USE="XMPP"

Per-application useflags, on the other hand, are defined in the /etc/portage/package.use file and only apply to specific packages. This allows you to enable or disable features on a package-by-package basis. For example, if you wanted to enable XMPP support only for the Pidgin instant messaging client, you could add the following line to package.use:

    net-im/pidgin XMPP

By using global and per-application useflags, you have more control over the features that are enabled in the packages that are installed on your system. This can be useful if you want to optimize your system for specific tasks or if you want to avoid compiling in features that you don't need.

In terms of my setup, I decided to stick with a tiling window manager (dwm) and a compositor (picom) rather than using a full-fledged desktop environment. I did this for aesthetics and because I find it to be a more efficient way to work.

Overall, I've been really happy with my switch to Gentoo. It's allowed me to customize my system to my liking and I've been able to get a bit more experience with different init systems. Plus, I love the tiling window manager setup.

If you're considering making the switch to Gentoo, I'd definitely recommend giving it a try! Just be prepared to put in a bit of time to set it up and customize it to your liking.
